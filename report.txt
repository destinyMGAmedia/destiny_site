Church Weekly Reporting Module PRODUCT REQUIREMENTS DOCUMENT (PRD)
Organization: Destiny Mission Global Assembly
 Module: Weekly Assembly Reporting System

1. Overview
This module will digitize the Church Weekly Report Template into an online system where each assembly submits:
Weekly Administrative Report
Weekly Financial Report
All data will be stored centrally and accessible to the Global Administrator via a dashboard.

2. Core Structure (2 Main Sheets → 2 Web Sections)
The system will have 2 main sections (tabs/pages):
Weekly Administrative Report
Weekly Financial Report

3. System Features (Important for Developer)
3.1 Auto Calculations
Attendance totals
Income totals
Expense totals
Net balance

3.2 Validation Rules
Required fields must be filled before submission
Numbers only in financial fields
No negative values

3.3 Submission Flow
User fills Administrative Report
User fills Financial Report
Click Submit Weekly Report
Data saved in database
Confirmation message displayed

3.4 Admin Dashboard
Global Admin should be able to:
View all reports by:
Week
Assembly
Filter and search
Download reports (PDF/Excel)
See summaries:
Total income per week
Attendance trends
Assemblies that have not submitted

4. Data Relationships
Each report is linked by:
Branch Name
Reporting Week
👉 This ensures Admin + Financial reports stay connected

5. User Roles
Branch Admin
Submit weekly reports
Global Admin
View all reports
Monitor performance
Export data

6. Non-Functional Requirements
Mobile-friendly (many pastors will use phones)
Simple UI (non-technical users)
Secure login system
Fast submission (low data usage preferred)

7. Key Improvement (Very Important)
👉 Add Submission Status Tracking
Each branch should show:
✅ Submitted
❌ Not Submitted
This helps leadership quickly follow up.

8. Future Enhancements
WhatsApp reminders for weekly submission
Auto-generated monthly reports
Graphs (attendance & finance trends)
Multi-level admin access
